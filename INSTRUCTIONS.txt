Deployment Guide for Hosting the IMS Application

1.  Uploading Files to Hosting

1.1 Install Dependencies - Navigate to the IMS project folder. 
- Run:
npm install

1.2 Build the Frontend 
- Navigate to the frontend directory. 
- Run:
npm run build 
- Upload only the contents of the dist folder to hosting.

1.3 Prepare Files for Upload 
- You may delete all generated QR codes and barcodes, but not their folders. 
- Upload the .htaccess file to prevent CORS issues.

1.4 Backend Configuration 
- In backend/conn.php, update: 
- $dsn 
- BASE_STORAGE_PATH 

- In backend/email_config.php, update: 
- $domain (set to hosting domain)

1.5 Required Folders to Upload 
- backend/
- vendor/ 
- dist/ (or its extracted files)

2.  Database Creation and Population

2.1 Create Database Schema - Open table.sql. - Copy all statements
EXCEPT: CREATE DATABASE IF NOT EXISTS IMS; USE IMS; - Paste into
phpMyAdmin SQL tab and execute.

2.2 Insert Initial Admin Test Data 
- Open admin_testing_population.sql. 
- Copy all statements. 
- Paste into phpMyAdmin SQL tab and execute.

3.  Uploading Initial Assets

3.1 Serviceable Assets 
- File: to_uploads/assets-format.xlsx 
- Follow
the format and convert to CSV. 
- Import via Serviceable Asset Module → Import Assets.

3.2 Consumables 
- File: to_uploads/consumables-format.xlsx 
- Convert to CSV. 
- Import via Available Consumables Module → Import Consumables.

3.3 Rooms 
- File: to_uploads/room-format.xlsx 
- Convert to CSV. 
- Import via Room List Module → Import Room.

3.4 Branding Header/Footer Image 
- File: to_uploads/kld-format-image.png 
- Upload via Settings Module → Branding Settings.
